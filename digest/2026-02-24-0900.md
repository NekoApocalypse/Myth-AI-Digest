# Daily AI Digest — Tuesday, February 24, 2026

> **Headline:** An AI agent wrote a working FreeBSD kernel Wi-Fi driver from scratch — not by hacking at existing code, but by generating a full spec first, proofreading it with a second agent, then building clean-room from the spec. This is what agentic software engineering actually looks like when it works.

---

## The Stack

**[Agents]** **AI-written FreeBSD Wi-Fi kernel driver (BCM4350)** — A spec-first, multi-agent pipeline (Pi + Claude Opus/Codex/Gemini) produced a working FreeBSD kernel module for a chip with no native driver; the author wrote zero code. The takeaway isn't "AI wrote a driver" — it's that decomposing the task into *spec generation → spec verification → clean-room implementation* is what unlocked success where a naive "just port it" prompt failed. [Source](https://vladimir.varank.in/notes/2026/02/freebsd-brcmfmac/) | [GitHub](https://github.com/narqo/freebsd-brcmfmac)

**[Agents]** **Wolfram CAG — MCP service + Agent One API** — Wolfram is shipping Computation-Augmented Generation (CAG): an MCP endpoint, a drop-in LLM API with Wolfram baked in, and fine-grained component APIs. Unlike RAG, CAG computes results live rather than retrieving cached text, which means LLM agents get verifiable symbolic math, curated scientific data, and structured reasoning on demand. If you're building agents that need to be *correct* rather than just *fluent*, this is the integration to watch. [Source](https://writings.stephenwolfram.com/2026/02/making-wolfram-tech-available-as-a-foundation-tool-for-llm-systems/)

**[Hardware]** **ASML EUV light source advance — +50% chip yield by 2030** — Content unavailable (paywalled), but the headline signal is clear: if ASML's EUV throughput increases 50%, the cost-per-chip for AI accelerators drops materially before the decade ends. This is the upstream constraint on everything else in the stack.
