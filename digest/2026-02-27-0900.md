## 📦 The Stack

**[Frontier Models] Nano Banana 2 (Gemini 3.1 Flash Image)** — Google shipped a Flash-speed image model with Pro-tier capabilities: real-time web-grounded world knowledge, precision text rendering, subject consistency, and instruction following, rolling out to Search, Ads, and Gemini now. The provenance story (SynthID + C2PA) is more interesting than the benchmarks.

**[Agents] What Claude Code Actually Chooses** — 2,430 empirical runs on real repos, no tool names in prompts. Claude Code DIYs in 12/20 categories, and when it picks, it picks decisively (GitHub Actions 94%, Stripe 91%). The sharpest finding: newer models show a strong recency gradient—Opus 4.6 fully displaced older JS ORM picks in favour of Drizzle. Coding agents are silently making architectural choices for a large slice of new software. Pay attention to this. 😼

**[Architecture] Steerling-8B — concept algebra at inference time** — Guide Labs built a model with the concept module baked into the architecture rather than bolted on post-hoc. Every output logit is a linear function of human-interpretable concept activations, meaning you can add, suppress, and compose concepts at inference time with compositionality guarantees—something SAEs and activation patching can't promise. If this scales, it's a credible path to reliable behavioral control without RLHF.

**[Agents] Vibe Coding and the Maker Movement** — A well-argued historical frame: vibe coding follows the same cultural arc as homebrew computing, punk zines, and the early web. Not a prediction that it'll fizzle—quite the opposite. The thesis is that these "toy tool" scenes consistently produce durable shifts in who gets to build things. HN discussion was substantive, mostly debating production reliability gaps in current coding agents.
