## The Stack

[Hardware] **Mesh LLM — Distributed LLM inference on iroh** — Pools GPUs and memory across machines into a single OpenAI-compatible endpoint. Serves 40+ models from 0.5B to 235B MoE, with pipeline splitting for models too big for any single box. Built on iroh's P2P QUIC transport (authenticated, NAT-traversing, no central server). Pluggable architecture: MCP, HTTP, gossip, all over the same connection. The protocol design is clean — 7 stream types demuxed by a single leading byte. If you have GPUs under desks and want them to behave like one compute pool, this is the right shape.
<https://www.iroh.computer/blog/mesh-llm>

[Hardware] **Reame — CPU-first inference server that gets faster as it runs** — Built on llama.cpp with the thesis: on CPU, never compute the same thing twice. Persistent disk KV cache (zstd, checksummed, survives restarts). N-gram generation archive (Palimpsest) that reuses past outputs. Self-regulating speculative decoding that turns itself off when it's counterproductive. Interleaved multi-user batching. The Conclave: majority-vote consensus across N candidates. Honest benchmarks — they publish the negative results where speculation hurts and Conclave doesn't close the gap to a bigger model. Runs Qwen2.5-7B at 3.3 tok/s on Oracle's free tier. 210 isolated test cases. This is engineering with receipts.
<https://github.com/swellweb/reame>

[Agents] **Sqlsure — Deterministic semantic checks for AI-generated SQL** — Catches fan-out double-counting, additivity violations, wrong join keys, and PII exposure before the query runs. Not an LLM. Rules are dictionary lookups, verdicts are deterministic and auditable. Audited 2,568 BIRD/Spider gold queries with zero false positives — and found a provably wrong gold answer (8x error). Ships with MCP server so your AI agent can draft → check → fix → check → execute. Uses your existing dbt tests and PK/FK declarations as the rulebook. In 0.1 ms. Apache-2.0.
<https://github.com/sqlsure/sqlsure>
