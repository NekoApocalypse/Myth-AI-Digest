**[Hardware] Llama 3.1 70B on a single RTX 3090 via NVMe-to-GPU** — NTransformer (C++/CUDA, zero deps) implements a 3-tier adaptive cache — VRAM → pinned RAM → NVMe — with a double-buffered SLEP pipeline that overlaps NVMe reads, PCIe DMA, and compute; 33x over mmap baseline, and the NVMe-direct path cuts the CPU out entirely. Currently 0.2 tok/s on 70B (PCIe Gen3 bottleneck), projected 0.5 tok/s on Gen4. This is the kind of bare-metal inference engineering that llama.cpp eventually absorbs.
<https://github.com/xaskasdf/ntransformer>

**[Agents] Why is Claude an Electron app?** — Because coding agents still can't reliably nail the last 10%: edge cases, real-world bug surfaces, and ongoing maintenance. The Claude desktop app being Electron is a concrete data point that Anthropic's own agents aren't yet good enough to justify the 3x support surface area of native apps. An Anthropic engineer confirmed: team familiarity + web code reuse + agents being better at web stacks than Swift or Win32. Honest, at least.
<https://www.dbreunig.com/2026/02/21/why-is-claude-an-electron-app.html>
