### Thinking Machines Lab Drops Inkling — 975B MoE, Full Weights, 1M Context

**HN 605pts** · [thinkingmachines.ai](https://thinkingmachines.ai/news/introducing-inkling/)

Encoder-free multimodal — audio via dMel spectrograms, vision via 40×40 pixel patches with lightweight hMLP embedding.

### Gemma 4 26B at 5 tok/s on a $300, 13-Year-Old Xeon

**HN 221pts** · [neomindlabs.com](https://www.neomindlabs.com/2026/06/08/running-gemma-4-26b-at-5-tokens-sec-on-a-13-year-old-xeon-with-no-gpu/)

Dual Xeon E5-2690 v2 (Ivy Bridge, 2013), DDR3, zero GPU. The ik_llama.cpp fork assumed AVX2 — this chip only has AVX1. The MoE FFN ops (`MOE_FUSED_UP_GATE`, `FUSED_UP_GATE`) compiled out silently, leaving 240 tensors per forward pass uninitialized. Output: fluent multilingual gibberish, deterministic, no NaNs.

The fix came from Claude instrumenting raw logits, spotting the +16 mean bias, and tracing it to the two missing dispatcher cases. Three commits: portable scalar fallbacks for quantize row ops, graph builder emits stock `mul_mat_id` + `fused_mul_unary` instead of the AVX2-only fused kernels, and CI stub cleanup. End result: 5.2 tok/s decode, ~16 tok/s prompt eval. The author's real point: AI-assisted debugging of performance-critical C++ is not a demo anymore — it's a workflow.

### Ambiance — A Unix-Philosophy Agent Harness

**HN 164pts, 82 comments** · [eardatasci.github.io](https://eardatasci.github.io/c/ambiance/index.html)

Maps agent infrastructure onto the Unix filesystem: agents are "users" (`/home`), tools are "binaries" (`/bin`), external data is "drivers" (`/sys`), logs in `/var`. An event-driven "kernel" watches the filesystem for changes and invokes LLMs with coalescence strategies — no polling, no heartbeat granularity.

Three built-in agents: `root` (system-level coding, driver/binary maintenance), `pai` (human-facing interaction), `librarian` (journals what works and what doesn't). Core thesis: the model's Unix priors are free — a harness built from files, users, and text streams will always beat one the model has to be taught. Debatable but internally consistent, and the implementation direction is cleaner than most "agent OS" projects.

### Designing APIs for Agents (Not Humans)

**HN 40pts** · [freestyle.sh](https://www.freestyle.sh/blog/opinion/designing-apis-for-agents)

Freestyle.sh's thesis, backed by shipping experience: agent-oriented API design inverts every human-oriented pattern.

- **Defaults are bad.** Agents can read your entire docs in one prompt. Make them fill every field explicitly — explicitness is cheap, debugging ambiguity is expensive.
- **Errors are not bad.** 27% of agent friction comes from errors, but only because errors are vague. A precise error is the best surface an agent has for recovering from a misconception. Don't coalesce, don't be "helpful" — fail loudly and tell them where the docs are.
- **Prefer specific field names.** `displayName` over `name`. `slug` over `id`. Agents will use `name` five different ways across ten use cases.
- **Facts, not feelings.** Your API's value is the fact it provides (bill paid, message sent, VM provisioned). SDKs that hide complexity are worse than guides that explain it.

They ripped out their entire declarative SDK abstraction (`new VmSpec({ with: { bun: new VmBun() } })`) and replaced it with raw `vm.exec()` + a guide doc. Cleaner onboarding, cleaner code, agents don't hallucinate wrappers.

## ⚡ Low-Level Inference

### libargus — Zero-Allocation LLM Runtime for Java via Panama FFM

**HN 26pts** · [github.com/projectargus-cc/libargus.cc](https://github.com/projectargus-cc/libargus.cc)

A unified C runtime wrapping llama.cpp + Whisper + libmtmd (multimodal), designed for zero-copy integration with JDK 22+ Project Panama FFM. Single process-global backend to prevent VRAM fragmentation across text/audio/vision subsystems. Pointers-only ABI — no pass-by-value, no C++ polymorphic boundaries. Zero JVM heap arrays on hot paths; all token tapes and audio buffers go through `MemorySegment` directly.

Supports speculative decoding, Multi-Token Prediction, KV cache quantization (Q8_0/Q4_0), embedding extraction, and video frame iteration via FFmpeg subprocess pipes. Clean Java API with `AutoCloseable` resources. The README is unusually thorough — if you need to ship LLM inference inside a JVM process without the usual JNI pain, this is the cleanest path I've seen.
