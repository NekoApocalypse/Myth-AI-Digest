## The Stack

**[Frontier Models] Unlimited OCR — Baidu open-sources a one-shot long-horizon OCR model** (432 pts)
Baidu drops an open-weight model that pushes past DeepSeek-OCR with multi-page, PDF-native long-horizon parsing. Ships with SGLang serving, 32K context, and a gundam/basic dual inference mode. The implication: long-form document digitization pipelines that previously needed chained OCR + layout analysis can now be done in a single pass. Code + weights on HF. <https://github.com/baidu/Unlimited-OCR>

**[Hardware] FUTO Swipe — the first actually useful open on-device swipe model** (229 pts)
Three-model ensemble (encoder + ContextLM + decoder) totaling ~2.5M params, runs in milliseconds on low-end devices. ~4% top-4 fail rate, under 1% excluding OOV. What matters: a fully local, GPL-licensed C++ inference library with dictionary-constrained beam search, meaning custom keyboards on niche OSes or privacy-first devices no longer need to license GBoard's stack. <https://swipe.futo.tech/>

**[Agents] The Coming Loop — Armin Ronacher on harness-level agent loops** (308 pts)
Ronacher draws a sharp line between where outer-loop patterns work (porting, security scanning, experimentation) and where they don't (code meant to last). Core thesis: loops amplify LLMs' defensive-overcoding tendency — "make malformed states impossible" > "handle every malformed case," and loops teach the opposite. The "software as organism" analogy is the most honest framing I've seen of what full-auto coding actually implies. <https://lucumr.pocoo.org/2026/6/23/the-coming-loop/>
