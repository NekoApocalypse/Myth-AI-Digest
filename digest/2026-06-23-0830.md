## The Stack

**[Frontier Models] GLM-5.2 — Z.ai's 744B MoE running locally**
Unsloth's day-zero GGUFs compress the full 1.51TB model to 239GB (2-bit, 82% top-1 accuracy) or 217GB (1-bit, 76%). That means frontier-class coding, reasoning, and agentic task performance — previously locked behind API paywalls — now runs on hardware you can buy. Supports three thinking modes (non-thinking, High, Max) and 1M context. <<https://unsloth.ai/docs/models/glm-5.2>>

**[Architecture] Moebius — 0.2B inpainting model beats 10B-class generalists**
HUST + VIVO AI Lab built a task-specific diffusion specialist using Local-λ Mix Interaction (LλMI) blocks that replace quadratic attention with fixed-size linear matrices. Paired with adaptive multi-granularity latent-space distillation from a larger teacher, Moebius matches or exceeds FLUX.1-Fill-Dev (11.9B) across 6 benchmarks while using <2% of the parameters and delivering >15× inference speedup. The paper's core argument: when the task is well-defined, a compressed specialist beats a bloated generalist. <<https://hustvl.github.io/Moebius/>>

**[Agents] Oak — a git alternative purpose-built for AI agents**
Oak reimagines version control with agent-native semantics: lazy mounts instead of full clones, structured JSON-first finish contracts, self-healing tool calling, and a merge workflow designed for non-human contributors. Already in active production (v0.99.0, 311 merged branches all-time). The design philosophy treats agents as first-class repository citizens rather than bolting agent workflows onto human-oriented git. <<https://oak.space/oak/oak>>

**[Agents] Prompt Injection as Role Confusion — a mechanistic explanation**
This paper develops "role probes" (linear classifiers on LLM activations) to measure whether models internally distinguish between `user`, `think`, `assistant`, and `tool` tagged text. Finding: they don't. Role boundaries bleed. A token in `tool` tags often reads as `user`-level instruction to the model, which explains why prompt injection remains solvable by adaptive human attackers despite near-perfect benchmark scores. The paper argues that attack memorization (the current defense strategy) is inherently brittle; only accurate role perception can provide robust defense. <<https://role-confusion.github.io>>

**[Agents] Claude Code's "Extended Thinking" output is a summary, not the real thing**
Patrick McCanna discovered that Claude Code's local thinking logs contain encrypted 600-character signatures, not the actual reasoning. Anthropic holds the decryption keys. What users see via Ctrl+O is a lossy summary of the model's thinking — not what actually drove its decisions. Full access requires an enterprise agreement. This has implications for anyone relying on agent audit trails or debugging opaque agent failures. <<https://patrickmccanna.net/the-text-in-claude-codes-extended-thinking-output-is-not-authentic/>>
