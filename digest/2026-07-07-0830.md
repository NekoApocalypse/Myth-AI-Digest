## The Stack

**[Architecture] A Global Workspace in Language Models**
Anthropic's interpretability team discovered J-space using a Jacobian lens — for every word in Claude's vocabulary, they found the activation pattern that makes that word more likely to appear. These patterns are causally editable (inject "rugby," the model reports thinking about rugby), reportable, and mediate multi-step reasoning silently. When J-space is suppressed, Claude still talks fluently but loses higher-order cognition. It was not designed — it emerged during training. Paper, code, and Neuronpedia demo linked. <https://www.anthropic.com/research/global-workspace>

**[Frontier Models] GLM 5.2 and the Coming AI Margin Collapse**
GLM 5.2 from Z.ai hits genuine Opus/GPT quality at $4.40/MTok — less than 20% of frontier pricing. The killer feature: drop-in API compatibility (both OpenAI and Anthropic endpoint formats). The author swapped it into Claude Code and couldn't tell the difference for non-interactive agentic tasks. Weaknesses: no vision, slower due to excessive thinking, poor web search. But at maybe 50%+ cheaper for equivalent quality, the switching cost from the $25/MTok tier is near zero. Margin collapse thesis is no longer theoretical. <https://martinalderson.com/posts/the-upcoming-ai-margin-collapse-part-1-glm-5-2/>

**[Agents] OfficeCLI — Office Suite for AI Agents**
Single binary, open source, no Office installation required. Gives AI agents full read/write control over .docx, .xlsx, and .pptx via CLI with built-in HTML rendering for the agent's visual feedback loop. One-line curl install teaches Claude Code, Cursor, Copilot, etc. the commands automatically. This is infrastructure for the agent economy — plugging the last major file-format gap. <https://github.com/iOfficeAI/OfficeCLI>

**[Hardware] Ternlight — 7 MB Embedding Model in Your Browser**
WASM-based embedding model, ~5ms inference, zero server dependency. Ships as a single npm package. Enables fully client-side semantic search with three lines of code. The 5MB "mini" variant is already powering a React docs search demo. Tiny, fast, local, free — the exact shape of the post-API embedding future. <https://ternlight-demo.vercel.app/>

**[Agents] Pruning RAG Context by 68% with 96% Recall**
kapa.ai inserted a small cheap LLM between retriever and generator as a listwise pruner. The key insight: relevance is a property of the chunk *set*, not individual chunks — a pointwise reranker can never catch that chunk B is only useful alongside chunk A. Their 5-level grading scale (Essential → Unrelated) judged by an LLM that sees everything together drops two-thirds of context while keeping almost all needed information. Net query cost cut: ~33%. <https://www.kapa.ai/blog/how-we-prune-rag-context>
