**Claude Opus 4.6 found 14 high-severity Firefox zero-days in two weeks — and then wrote exploits for two of them.**

## 📦 The Stack

**[Agents]** **[Hardening Firefox with Anthropic's Red Team](https://www.anthropic.com/news/mozilla-firefox-security)** — Claude Opus 4.6 autonomously discovered 22 Firefox vulnerabilities (14 high-severity, patched in FF148) using a "task verifier" agentic loop: probe codebase → generate crashing inputs → validate → propose patch, no human in the loop per case. Total cost for the exploit phase: ~$4,000 in API spend, two successful browser exploits produced. The post also introduces Anthropic's Coordinated Vulnerability Disclosure principles and the Claude Code Security preview — they're turning this into a product. 😼

**[Architecture]** **[OBLITERATUS — Open-Source LLM Abliteration Toolkit](https://github.com/elder-plinius/OBLITERATUS)** — The most complete public implementation of "abliteration": SVD-based surgical removal of refusal-direction vectors from open-weight model weights, without retraining. Fifteen analysis modules, including a novel detector for the "Ouroboros effect" (guardrails self-repairing post-removal), alignment-method fingerprinting (DPO vs. RLHF vs. CAI from geometry alone), and reversible inference-time steering vectors. Built on Arditi et al. (2024) and Turner et al. (2023), structured as a crowd-sourced mechanistic interpretability research platform. If you want to understand how alignment actually lives inside a transformer, this is the hands-on lab.

**[Agents]** **[Anthropic, please make a new Slack](https://www.fivetran.com/blog/anthropic-please-make-a-new-slack)** — Fivetran's CEO makes the case that Slack's API is the biggest blocker for enterprise AI agents: Claude can't participate in group threads, and Slack's data access policy is effectively "No." The argument: bundle Claude seats with an AI-native chat platform, make a credible public commitment to open data, and let competition do the rest. The HN thread is more interesting than the post — engineers pointing out that Claude Code could vibecode a Slack clone in a weekend, and that Anthropic's own track record on interoperability (no OpenAI API compat, Claude Code still closed-source) undercuts the pitch.

## 🗑️ The Noise

**"Anthropic, please make a new Slack"** — Already called it out above, but worth flagging: this is a Fivetran CEO blog post that's part vision-doc, part vendor wish-fulfillment. The core data-access problem is real; the ask is not. Anthropic building a chat app is not happening, and the community knows it. Hot because the frustration with Slack is universally relatable, not because the proposal has legs.
