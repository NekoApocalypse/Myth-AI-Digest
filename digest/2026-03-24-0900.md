# Daily AI Digest — 2026-03-24

## 🔺 The Headline

**iPhone 17 Pro runs a 400B MoE LLM on-device** — and while 0.6 t/s isn't usable, the Flash-MoE technique (stream weights from NVMe on demand, activate only 4-10 of 512 experts per layer) just proved that frontier-scale inference no longer requires datacenter RAM.

## 📡 The Stack

**[Hardware] iPhone 17 Pro Demonstrated Running a 400B LLM** — Open-source Flash-MoE project (@anemll) demonstrates a 397B parameter MoE model running on 12GB of device RAM by streaming expert weights from flash storage on demand; M5 Pro's doubled SSD bandwidth is what makes this barely viable. Not usable at 0.6 t/s, but the architecture path is confirmed. [(474pts)](https://news.ycombinator.com/item?id=47490070)

**[Agents] Autoresearch on an Old Research Idea** — Developer runs Claude Code in a locked Docker container on a loop (edit `train.py` → run evals → record → repeat), applied to a medical imaging model on an Ukiyo-e dataset; the agent fixed a temperature-clamp bug and ran Optuna-style hyperparameter search, improving the benchmark. Key finding: LLM agents are currently good at bug-finding + HPO, but weak at novel architectural exploration. [(261pts)](https://news.ycombinator.com/item?id=47493460)

**[Agents] How I'm Productive with Claude Code** — Engineer at Tano documents a concrete agentic coding workflow: custom `/git-pr` skill, SWC for sub-second rebuilds, Claude Code's visual preview for agent self-verification, and a port-allocation system enabling 5 parallel worktrees without collision. More interesting as infrastructure pattern than productivity brag. [(107pts)](https://news.ycombinator.com/item?id=47494890)
