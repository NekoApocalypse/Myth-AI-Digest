## The Stack

**[Agents]** [Leanstral: Open-Source Agent for Trustworthy Coding and Formal Proof Engineering](https://mistral.ai/news/leanstral) *(277 pts)* — Mistral releases Leanstral, a 120B sparse model (6B active) trained specifically for Lean 4 formal proof engineering; it outperforms Sonnet 4.6 on FLTEval benchmarks (26.3 vs 23.7 at pass@2) at 15× lower cost ($36 vs $549 per run), with Apache 2.0 weights and MCP support. The architecture bets on parallel inference with Lean as a perfect verifier rather than bigger models, which is a cleaner scaling story than brute force.

**[Hardware]** [NVIDIA Launches Vera CPU, Purpose-Built for Agentic AI](https://nvidianews.nvidia.com/news/nvidia-launches-vera-cpu-purpose-built-for-agentic-ai) *(121 pts)* — NVIDIA's Vera CPU delivers 2× efficiency and 50% faster throughput vs traditional rack CPUs, with 1.8 TB/s NVLink-C2C coherent bandwidth to Rubin GPUs; a single Vera rack runs 22,500 concurrent CPU environments. Alibaba, ByteDance, Meta, Oracle, CoreWeave, Dell, HPE, and Lenovo are already on board — this is the infrastructure bet for agentic AI at scale, not just a GPU companion chip.

**[Agents]** [Apideck CLI: Your MCP Server Is Eating Your Context Window](https://www.apideck.com/blog/mcp-server-eating-context-window-cli-alternative) *(116 pts)* — A rigorous breakdown of MCP's context overhead problem: 40 tools ≈ 55K tokens burned before the agent reads a single message; Scalekit benchmarks show MCP costs 4–32× more tokens than equivalent CLI calls for the same tasks. The proposed fix — a CLI with progressive disclosure — is pragmatic, though it trades structured typing for token efficiency; worth reading if you're hitting context limits in production agent pipelines.

**[Agents]** [Show HN: Godogen — Claude Code Skills That Build Complete Godot 4 Games](https://github.com/htdt/godogen) *(150 pts)* — Two-skill Claude Code pipeline (planner + executor) that generates full Godot 4 projects from a text description: GDScript writing, Gemini-powered asset generation, Tripo3D image-to-3D conversion, and visual QA via screenshot analysis. Runs on commodity hardware; most interesting as a demonstration that multi-agent skill composition can close the loop on creative software generation end-to-end.

**[Figures]** [US Job Market Visualizer — Karpathy](https://karpathy.ai/jobs/) *(397 pts)* — Karpathy builds a treemap of 342 BLS occupations (143M jobs) with LLM-powered custom coloring, including a "Digital AI Exposure" scorer with a published prompt. The open-source pipeline (scraper → LLM scorer → treemap) is the real contribution; anyone can swap in a different prompt ("humanoid robotics exposure", "offshoring risk") and re-run. Low-key infrastructure for economic scenario analysis.

**[Architecture]** [Language Model Teams as Distributed Systems](https://arxiv.org/abs/2603.12229) *(64 pts)* — Proposes using distributed systems theory as a principled framework for LLM team design — when to use multiple agents, how many, what structure — drawing explicit parallels to classic distributed computing tradeoffs. Timely theoretical scaffolding for a space currently running on vibes and trial-and-error.

**[Hardware]** [Locally Hosted Voice Assistant — Full Engineering Journey](https://community.home-assistant.io/t/my-journey-to-a-reliable-and-enjoyable-locally-hosted-voice-assistant/944860) *(310 pts)* — Detailed practitioner writeup on building a reliable local voice stack: open-source wake word detection, local STT/TTS, Home Assistant integration, and the persistent unsolved problem of wake word sensitivity vs. false positives. The community engagement (92 comments) is high — this is the current frontier for on-device voice inference from non-researchers.
