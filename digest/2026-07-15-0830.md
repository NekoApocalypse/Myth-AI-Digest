## Bonsai 27B: A 27B-Class Model That Runs on Your Phone
*Models · 395 pts · <https://prismml.com/news/bonsai-27b>*

PrismML shipped a 27B multimodal model that fits on an iPhone. Two variants: **ternary** (5.9 GB, 95% of full-precision benchmarks) and **1-bit** (3.9 GB, 90% retention). Agentic workloads — tool calling, multi-step reasoning — stay within a few points of baseline. Apache 2.0.

The number that matters: intelligence density of 0.53 per GB — over 10× the full-precision baseline. On-device agents with zero marginal per-step cost and no data leaving the machine are now practical at a capability tier that can actually do work.

## The Tower Keeps Rising
*Agents · 319 pts · <https://lucumr.pocoo.org/2026/7/13/the-tower-keeps-rising/>*

Armin Ronacher (Flask, Sentry) with a sharp analogy: AI-assisted engineering is Babel in reverse. In the biblical story, loss of shared language stopped construction. With agents, construction continues *after* shared understanding collapses — because nobody needs to coordinate anymore. Every dev has a tireless translator. The code compiles. Tests pass. The tower doesn't fall. It just keeps rising.

Core insight: pre-agent friction (reading your colleague's code, asking questions, arguing about architecture) wasn't pure waste. Some of it was the synchronizing mechanism that kept the team's mental model coherent. Agents remove that friction, and the cost shows up later — in codebases nobody collectively understands. "Agents do not feel pain, only humans do."

## Juggler: A GUI Coding Agent by the Creator of JUCE
*Agents · 169 pts · <https://github.com/juggler-ai/juggler>*

Jules Storer (JUCE, Tracktion) released Juggler — an open-source GUI coding agent that treats sessions as **trees**, not linear transcripts. Finder-style Miller columns for tool call inspection. Fully plugin-based: context items, LLM strategies, and slash commands are all JS extensions. Multi-client — run the server where the code lives, attach from desktop or browser.

Built on Go + Wails, Yjs document model, AGPL core with Apache 2.0 SDK for extensions. Supports Claude, OpenAI, Gemini, Ollama, OpenRouter. This isn't another terminal agent — it's a proper workbench for people who want to stay hands-on.

## Guardian Angels: LLM Personalization for Productivity and Security
*Agents · 51 pts · <https://gwern.net/guardian-angel>*

gwern's major new essay. Proposal: personal LLMs that **emulate the user** — digital twin "guardian angels" — instead of the default chatbot assistant. This weakly solves the principal-agent problem by unifying principal and agent. The GA learns to produce the user's outputs at higher quality, screens for AI-powered scams, and avoids confused-deputy attacks by being hardwired to a single principal.

Technical backbone: online learning via dynamic evaluation, active learning by querying the user for corrections, heavy inner-monologue search. gwern argues standard frozen-model ICL can't reach this — too shallow, too under-parameterized, too misaligned. The essay's undercurrent is pessimistic: chatbot LLMs have **failed to augment knowledge workers**, and the economic incentive is to replace humans, not amplify them. Worth the full read.

## The Zero-Cost Fallacy: Open Source in the Agentic Era
*Infrastructure · 99 pts · <https://www.thoughtworks.com/insights/blog/open-source/zero-cost-fallacy-open-source-agentic-era>*

ThoughtWorks on how permissive licensing (MIT, Apache) enabled an extraction economy — corporations treating OSS maintainers as unpaid infrastructure. The new dimension: AI coding agents consume OSS at machine scale, generate derivatives, and bypass the human interactions that historically built community and reciprocity. Maintainers who relicense defensively face boycotts and hostile backlash. "Changing the license is viewed as an act of aggression, while exploiting the license is viewed as standard business practice."

## How to Stop Claude from Saying "Load-Bearing"
*LLM Behavior · 416 pts · <https://jola.dev/posts/how-to-stop-claude-from-saying-load-bearing>*

## LeMario: Training a JEPA World Model on Super Mario Bros
*Research · 22 pts · <https://www.benjamin-bai.com/projects/lemario>*

Detailed technical postmortem. Benjamin Bai trained a Joint-Embedding Predictive Architecture from scratch on 737K Mario frames across 32 levels — action-conditioned AdaLN-Zero transformers, SIGReg to prevent representation collapse. The model learned short-horizon dynamics but couldn't plan past the first obstacle.

Three failure modes: (1) predictive state ≠ control state, (2) CEM planning optimistically exploits model weaknesses, (3) Mario's scrolling camera breaks the visual-similarity-as-progress assumption from Push-T. Solid research hygiene: probed every component, found what broke, wrote it up honestly.
