## The Stack

[Figures] **When AI Builds Itself** — Anthropic Institute drops internal data: Claude authors >80% of merged code as of May 2026, typical engineer output up 8×, and Mythos Preview autonomously recovered 97% of a human-week AI safety research task in $18K of compute. [Agents] **Anthropic's defending-code-reference-harness** — Open-source reference pipeline for autonomous vulnerability discovery: recon→find→verify→dedupe→report→patch, each stage running sandboxed Claude agents inside gVisor with egress locked to the Claude API. Ships Claude Code skills (`/threat-model`, `/vuln-scan`, `/triage`, `/patch`) and a customizable harness for C/C++ ASAN-based memory bug hunting. Built from real security team engagements post-Mythos Preview.
<https://github.com/anthropics/defending-code-reference-harness>

[Hardware] **KVarN — Huawei CSL's KV-cache quantization vLLM fork** — Hadamard rotation + iterative variance normalization (Sinkhorn-like) before quantization gets you 3-5× more context and FP16-or-better throughput at 4-bit keys / 2-bit values. No calibration. One flag. Beats TurboQuant which required a 40-52% throughput sacrifice for similar capacity gains. If you're running long-context agents on vLLM, this is an immediate drop-in.
<https://github.com/huawei-csl/KVarN>

[Architecture] **Do Transformers Need Three Projections? (ICML 2026)** — Systematic study showing Q-K=V projection sharing gives 50% KV cache reduction with only 3.1% perplexity degradation. Stack it with GQA-4 and you hit 87.5% reduction; with MQA, 96.9%. Orthogonal to and composable with existing head-sharing methods. Direct path to practical on-device inference for large models. Code is public.
<https://arxiv.org/abs/2606.04032>
