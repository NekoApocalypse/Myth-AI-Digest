## 📦 The Stack

**[Agents] The Claude Code Source Leak** — Anti-distillation fake tool injection, server-side reasoning chain summarization (returns cryptographic summaries instead of raw outputs), and an `undercover.ts` with no force-off in external builds. The anti-distillation mechanisms are trivially bypassed (MITM proxy or one env var), but the undercover mode is a one-way door by design. Anthropic's second accidental exposure in a week; the model spec leak was days ago.
<https://alex000kim.com/posts/2026-03-31-claude-code-source-leak/>

**[Architecture] TinyLoRA — 13 Parameters to Reason** — Proposes sub-rank LoRA down to a single parameter; trains 8B Qwen2.5 to 91% GSM8K with 13 weights (26 bytes), recovering 90% of performance gains at 1000x fewer parameters. Critically: only works with RL training — SFT needs 100-1000x more params for the same effect. This is a strong signal that reasoning is encoded in a remarkably low-dimensional subspace, but only when shaped by RL.
<https://arxiv.org/abs/2602.04118>

**[Hardware] 1-Bit Bonsai (PrismML)** — 8B model in 1.15GB, 8× faster, 5× more energy-efficient than full-precision, matches leading 8B benchmarks. 1.7B hits 130 tok/s on an iPhone 17 Pro Max. Apache 2.0 (on the weights? unclear). If the benchmark claims hold under real workloads, this is the on-device inference story everyone's been waiting for.
<https://prismml.com/>

**[Frontier Models] Cohere Transcribe** — Open-weights 2B Conformer ASR, #1 on HuggingFace Open ASR Leaderboard at 5.42% avg WER, beating Whisper Large v3 and ElevenLabs Scribe v2. Apache 2.0, 14 languages, production-footprint-friendly. Solid drop-in for any speech pipeline.
<https://cohere.com/blog/transcribe>

**[Architecture] KV Cache Evolution: 300KB → 69KB per Token** — Solid technical writeup on the MHA → GQA → MLA progression with concrete byte costs. GPT-2 at 300KB/token, Llama 3 at 128KB (GQA), heading toward sub-70KB. Good mental model calibration if you're thinking about long-context inference costs.
<https://news.future-shock.ai/the-weight-of-remembering/>
