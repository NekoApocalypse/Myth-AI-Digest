**[Hardware]** **Google TPU 8t / TPU 8i** — Google's 8th-gen TPUs split into two purpose-built chips: TPU 8t for training and TPU 8i for low-latency inference, both co-designed with DeepMind specifically to handle the iterative, stateful demands of agentic workloads; GA later this year.
→ <https://blog.google/innovation-and-ai/infrastructure-and-cloud/google-cloud/eighth-generation-tpu-agentic-era/>

**[Agents]** **Parallel Agents in Zed** — Zed ships a Threads Sidebar that lets you run multiple agent threads simultaneously across different repos/worktrees in a single window, with per-thread agent and context isolation; open-source, 120fps, BYO agent.
→ <https://zed.dev/blog/parallel-agents>

**[Agents]** **Over-editing: Coding Models Are Doing Too Much** — Empirical study defines and benchmarks the "over-editing" failure mode (functionally correct but structurally divergent rewrites), finds it's invisible to test suites, and explores whether models can be trained to make minimal edits; code on GitHub.
→ <https://nrehiew.github.io/blog/minimal_editing/>

**[Breakout]** **Scoring Show HN for AI Design Patterns** — Deterministic CSS/DOM analysis of 500 Show HN pages quantifies the spread of LLM-generated design fingerprints (VibeCode Purple, colored left borders, Inter + Space Grotesk combos); Show HN submissions tripled post-Claude Code, prompting HN moderator restrictions.
