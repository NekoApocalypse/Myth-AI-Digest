## 📡 The Stack

**[Agents] Emdash — Open-source agentic dev environment for running 21+ coding agent CLIs in parallel, each in its own git worktree, locally or over SSH.**
This is what the multi-agent workflow actually looks like in practice: provider-agnostic, worktree-isolated, SSH-capable, ~500ms task startup. YC W26. It's an AGENTS.md situation in the best possible way.
🔗 <https://github.com/generalaction/emdash>

**[Architecture] Mercury 2 — Diffusion-based reasoning LLM at 1,009 tok/s on Blackwell, $0.25/$0.75 per 1M I/O.**
Autoregressive decoding is the bottleneck everyone's been politely ignoring. Mercury 2 generates via parallel refinement — decoupling reasoning quality from latency in a way that actually matters for agentic loops. The speed-quality tradeoff is real and the pricing is competitive. Watch this architecture.
🔗 <https://inceptionlabs.ai/blog/introducing-mercury-2>

**[Frontier Models] Moonshine Voice — Open-weights, on-device STT that beats Whisper Large V3 accuracy, scales down to 26MB, ships on iOS/Android/Pi.**
This is how on-device AI should work: no cloud dependency, low latency streaming, multilingual, cross-platform with batteries included. The libtech for voice-native agents just got a lot cheaper.
🔗 <https://github.com/moonshine-ai/moonshine>

**[Agents] Hugging Face Skills — Cross-agent skill packages interoperable with Claude Code, Codex, Gemini CLI, and Cursor.**
HF shipping a standardized skill format that works across competing agent CLIs is quietly load-bearing infrastructure. It's the first serious attempt at agent-skill portability — SKILL.md / AGENTS.md / gemini-extension.json all covered. The HF-specific skills (Hub ops, dataset management, model training) give agents authoritative workflows for the most common ML tasks.
🔗 <https://github.com/huggingface/skills>

**[Figures] METR abandons their AI dev productivity study — because devs refuse to work without AI anymore.**
Their 2025 finding of a 19% slowdown is now suspected to be selection bias: 30–50% of developers in the follow-up refused to submit tasks they didn't want to do without AI. The methodological failure *is* the signal. A controlled "no-AI baseline" is no longer obtainable in the wild.
🔗 <https://metr.org/blog/2026-02-24-uplift-update/>
