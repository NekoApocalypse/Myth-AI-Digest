## The Headline

**GPT-5.5 may have a hidden reasoning-token budget cap at exactly 516 tokens**, and the data is hard to ignore. 82% of all exact-516 reasoning-token events across Codex come from GPT-5.5 despite it being only 19% of total responses. The clustering spiked from near-zero in Feb to 53% of all GPT-5.5 runs in May. Meanwhile, mean reasoning tokens halved over the same window. Whether this is intentional throttling, a degraded fallback tier, or a scheduler bug — 44% of GPT-5.5 responses terminating at exactly 516 looks a lot less like a distribution and a lot more like a kill switch.

## The Stack

**[Frontier Models] GPT-5.5 Codex reasoning-token clustering at 516/1034/1552** — Community analysis of 390K+ Codex token_count records reveals GPT-5.5 disproportionately terminates reasoning at exact boundaries of 516, 1034, and 1552 tokens. The 516 → ≥516 ratio is 33.6× higher than the non-GPT-5.5 baseline, and this pattern coincides with a sharp drop in mean reasoning-token intensity from ~268 (Feb) to ~107 (May). The clustering is model-specific — GPT-5.2 and Codex variants show negligible rates. If this is an intentional reasoning budget, it's undocumented. If it's a bug, it's degrading complex-task performance silently. <https://github.com/openai/codex/issues/30364>

**[Agents] Claude Code potential session/cache leakage between Enterprise ZDR workspace instances** — A user on Anthropic's Enterprise ZDR plan reports Claude Code suddenly pivoting to discussing Minecraft temple construction mid-session, including specific brick-type questions, with no prior context. Multiple HN commenters corroborate cross-session contamination patterns across both Claude and Gemini — one describes API gateway HTTP desync/request-smuggling as the likely root cause (an off-by-one response routing bug seen across "$1T+ companies"). Comments also surface that the bug may be a hallucination triggered by a tool result mentioning `minecraft.py` (from Pygments lexer), but the OP reproduced it in a separate fresh session. Either way: Enterprise ZDR isolation guarantees deserve scrutiny. <https://github.com/anthropics/claude-code/issues/74066>

**[Breakout] AI-built PHP engine in Rust passes 17.4% of upstream PHP test suite, renders WordPress** — A developer who explicitly does not know Rust used an LLM to build a from-scratch PHP interpreter (~24K lines) validated entirely against PHP's own 22,000-test corpus. The core insight: don't let the AI grade its own homework. Every commit is gated by the upstream PHP test suite, with a public scoreboard. The engine now bootstraps WordPress, renders wp-admin, and has a bytecode VM running micro-benchmarks at 1–3× PHP 8.5. The experiment is less about AI capability and more about methodology: an external oracle (the test suite) makes the human the verifier rather than the implementer. <https://ekinertac.com/blog/i-dont-know-rust-my-ai-is-rewriting-php-in-it/>
