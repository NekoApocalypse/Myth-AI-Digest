## 📦 The Stack

**[Hardware/Inference]** **Ask HN: Has anyone replaced Claude/GPT with a local model for daily coding?** — The thread is a goldmine of practical signal: Qwen3.6 35B-A3B (3B active params via MoE) on Mac Studio 128GB or AMD Strix Halo with llama.cpp Vulkan is achieving ~5x coding speedup vs baseline, compared to Opus's ~15x, but completely offline with no API spend. Key technical details: `preserve_thinking = true` in llama.cpp models.ini eliminates KV cache thrashing on long agentic turns; Vulkan outperforms ROCm for most setups; Q8 quant on the K side of the KV cache significantly reduces tool-call loops. The gap to frontier models is real but shrinking on constrained, well-specified tasks.
<https://news.ycombinator.com/item?id=48542100>

**[Agents]** **My Homelab AI Dev Platform** — A concrete architecture for safe agentic deployment: OpenCode runs as a persistent server on an isolated VM, clones repos, pushes feature branches, and never touches live services directly. Human reviews PRs; GitOps handles the rest. The design principle — small blast radius + forced PR review gate — is the pattern for running AI agents in any environment where you need auditability. Missing piece: Forgejo doesn't expose CI job logs via official API, breaking the feedback loop.
<https://rsgm.dev/post/ai-dev-platform/>
