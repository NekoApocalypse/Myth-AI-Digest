## The Stack

**[Figures] "Claws are now a new layer on top of LLM agents" (Karpathy)** — The tweet crystallised a category: personal, user-owned agents that run on a schedule, talk via messaging apps, and hold credentials on your behalf. HN lit up defining the term, with the most useful thread contribution being the "lethal trifecta" (private data + untrusted input + external comms = prompt injection waiting to happen). OpenClaw got namedroped extensively as the reference implementation, including in a thread where someone described it as "insane mad scientist territory" — which, honestly, fair. Security patterns discussed include OTP-gated destructive actions, zero-trust sub-agent isolation, and domain allowlists. The category is real; the security model is still unresolved. *HN: 189pts*

**[Hardware] NTransformer: Llama 70B on a single RTX 3090 via NVMe-to-GPU** — A weekend vibe-code project that actually works: streams 70B weights directly from NVMe to GPU, bypassing CPU/RAM entirely using a userspace NVMe driver via VFIO passthrough. Gets 0.5 tok/s (usable for batch) with Q4_K_M + layer-skip at 36 VRAM + 44 RAM. The 3-tier adaptive cache auto-sizes from `cudaMemGetInfo` + `/proc/meminfo`. Setup is gnarly (patch NVIDIA DKMS for kernel 6.12+, disable IOMMU, unbind your secondary NVMe from the kernel) and you will brick your install if you use your boot drive. But the existence proof matters: 70B local inference is no longer a compute-budget problem — it's a patience-and-SSD problem. BSD-2 licensed, zero external deps beyond CUDA. *HN: 93pts*

**[Hardware] zclaw: full AI assistant in ≤888 KiB on an ESP32** — 25 KiB of actual app logic; the rest is WiFi stack (367 KiB), TLS (123 KiB), and cert bundle (91 KiB). Supports Telegram, cron schedules, GPIO control, persistent memory, and Anthropic/OpenAI/OpenRouter. It's a thin client calling cloud APIs — not local inference — but it's the clearest proof yet that the "claw" harness is trivially small. The 888 KiB cap is a flex; one commenter pointed out the real headline is that there is almost no code here. Available on ESP32-C3/S3/C6, MIT licensed. *HN: 92pts*

**[Agents] "Why is Claude an Electron app?"** — dbreunig points out the irony: Anthropic ships a $20k AI-written C compiler demo but its consumer desktop app is a janky Electron bundle. Boris from the Claude Code team confirmed: shared codebase with web, team familiarity with Electron, and "Claude is great at it." Felix (who owns the Electron app, including Claude Cowork) noted it includes Rust, Swift, and Go components despite the web wrapper. The real signal from 282 comments: code agents are genuinely good at the first 90% — spec-driven, cross-platform work is still bottlenecked at the last-mile edge cases and multiplied support surface. This is a useful temperature check on where vibe-coding actually breaks down. *HN: 328pts*

**[Breakout] Uncovering insiders and alpha on Polymarket with AI** — Author (Peter Liu, building "Compound AI") used a multi-agent query pipeline over Polymarket's onchain data to surface accounts with statistically anomalous win rates — i.e., probable insiders. Best comment thread explores why copying insiders is harder than it sounds: they exhaust your price level before you can follow, and fresh-address usage defeats naive wallet clustering. More interesting than the demo is the sub-discussion: HN's institutional-equities crowd noting that wallet clustering by funding-source timing is still surprisingly effective even with mixer usage, and that adverse selection from insiders crowds out calibrated forecasters — the classic prediction market death spiral. Relevant for anyone building AI-driven financial intelligence pipelines. *HN: 127pts*
