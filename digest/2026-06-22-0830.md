# Daily AI Digest — 2026-06-22

## The Headline
**Claude demands your passport.** Anthropic's identity verification rollout (via Persona) combined with recent US export controls is triggering a mass exodus of non-US developers toward open models. The HN thread at 536 points is less about the ID check itself and more about the dawning realization that building on US-hosted frontier models is now a supply-chain risk. 🍿

## The Stack

[Frontier Models] **Identity verification on Claude** — 536 points, 486 comments. Anthropic now requires government-issued photo ID for certain Claude capabilities, using Persona Identities as the verification partner. The HN discussion is a torrent of international users announcing they're pulling their subscriptions and migrating to open models (DeepSeek, GLM 5.2, Mistral). The practical concern: if your business depends on a SaaS model that can be yanked by geopolitics overnight, you've already made the mistake. <https://support.claude.com/en/articles/14328960-identity-verification-on-claude>

[Frontier Models] **Apertus — Swiss sovereign AI foundation model** — 151 points. EPFL, ETH Zurich, and CSCS dropped a fully-open model at 8B and 70B scale. Open weights, open data, open training code, documented alignment — the whole stack. EU AI Act compliant from day one (PII removal, opt-out respect), trained on 1000+ languages, competitive with top open models at equivalent scale. Timing couldn't be better. <https://apertvs.ai/>

[Agents] **Recall — fully-local project memory for Claude Code** — 59 points. A plugin that solves the cold-start problem for agentic coding sessions: it captures session transcripts locally, runs TF-IDF + TextRank summarization (zero API calls, no model tokens), and produces a compact ~1-2K token context.md for resuming. Pure Python, offline, privacy-respecting. The kind of tool that should've been built-in. <https://github.com/raiyanyahya/recall>

[Hardware] **Occupancy math on the AMD MI355X (CDNA4)** — 43 points. Deep technical guide to GPU kernel occupancy from first principles on AMD's latest accelerator. The key finding: maximizing occupancy is usually wrong. The matrix core sustains ~97% peak throughput even at low wavefront residency when tuned for ILP. Solid resource for anyone writing GEMM kernels that aren't just cuBLAS wrappers. <https://indianspeedster.github.io/blog/occupancy-math-mi355x/>
